## Why Phase 0 Matters

Before touching AWS or DevOps tools, a **strong foundation** is mandatory.
Most production failures happen not because of cloud services, but because of **weak fundamentals**.

This phase is designed to make you **comfortable, confident, and job-ready** before moving forward.

---

## 1. Linux Fundamentals

Linux is the backbone of DevOps. Almost every server, container, and cloud service runs on Linux.

### Topics Covered
- File and directory commands (`ls`, `cd`, `cp`, `mv`, `rm`)
- File permissions and ownership (`chmod`, `chown`)
- Process management (`ps`, `top`, `kill`)
- Disk and memory basics (`df`, `du`, `free`)
- Networking basics (`ip`, `ping`, `netstat`, `ss`)

### Recommended Playlist
🎥 **Linux Zero to Hero**

## 2. Git Fundamentals

Git is the heart of modern DevOps workflows.
Every CI/CD pipeline depends on Git.

### Topics Covered
- Repository initialization and cloning
- Branching strategy (feature, release, hotfix)
- Merging vs Rebase (when and why)
- Pull Request workflow
- Handling merge conflicts

### Recommended Playlist
🎥 **Git Zero to Hero**  
https://www.youtube.com/playlist?list=PL9YnOxYKGRNOl0WRxCDV7AaZqoQGne929

## 3. Basic Networking

DevOps engineers must understand **how data flows** from users to servers.

### Topics Covered
- DNS (how domain names work)
- HTTP vs HTTPS
- TCP/IP fundamentals
- Ports and protocols
- Client-server communication

### Recommended Playlist
🎥 **Networking for DevOps Engineers**

## 4. Shell Scripting Basics

Automation starts with scripting.
Even simple scripts can save hours of manual work.

### Topics Covered
- Bash syntax and execution
- Variables and arguments
- Conditional statements
- Loops
- Writing reusable scripts

## Outcome of Phase 0

After completing this phase, you will:
- Feel confident working on Linux servers
- Understand Git workflows used in real teams
- Know how networking works behind applications
- Be ready to automate basic tasks

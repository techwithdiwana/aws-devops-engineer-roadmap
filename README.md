## Tech With Diwana — Zero to Job-Ready

This repository covers **everything required to become an AWS DevOps Engineer**.
No topic is skipped — from **fundamentals to real-world production DevOps practices**.

---

## How to Use This Roadmap
- Follow the phases sequentially
- Practice every service hands-on
- Build at least one end-to-end DevOps project on AWS before applying for jobs

---

## Complete Roadmap Phases

### Phase 0 — Prerequisites
Linux fundamentals, Git, Networking, Shell scripting

### Phase 1 — AWS Fundamentals
Global infrastructure, IAM, AWS CLI & SDK, Shared responsibility model

### Phase 2 — Compute Services
EC2, Auto Scaling, Load Balancers, Launch templates

### Phase 3 — Networking (VPC)
VPC, Subnets, Route Tables, IGW, NAT, Route 53

### Phase 4 — Storage & Databases
S3, EBS, EFS, RDS, DynamoDB, ElastiCache

### Phase 5 — CI/CD on AWS
CodeCommit, CodeBuild, CodeDeploy, CodePipeline, GitHub Actions

### Phase 6 — Containers & Kubernetes
Docker, ECR, ECS, EKS, Blue/Green & Canary

### Phase 7 — Infrastructure as Code
CloudFormation, Terraform, State management

### Phase 8 — Monitoring & Logging
CloudWatch, CloudTrail, X-Ray, Prometheus, Grafana

### Phase 9 — Security & Compliance
IAM advanced, Secrets Manager, KMS, GuardDuty

### Phase 10 — Scaling, Reliability & Cost
HA, DR, Cost optimization, Well-Architected

### Phase 11 — Real-World DevOps Architecture
3-tier app, CI/CD rollback, Zero-downtime, Multi-account

---

## Must-Do Hands-on Projects
- EC2 + ALB + Auto Scaling deployment
- CI/CD pipeline using CodePipeline
- Containerized app on EKS
- Terraform-based infrastructure
- CloudWatch monitoring & alarms
- Secure AWS with IAM best practices
